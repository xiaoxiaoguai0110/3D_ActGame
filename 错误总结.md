# Git 推送错误总结

## 问题概述

将 Unity 项目推送到 GitHub 时遇到了一系列错误，最终通过只推送脚本文件解决。

---

## 错误一：推送被拒（non-fast-forward）

### 错误信息
```
! [rejected] main -> main (non-fast-forward)
```

### 原因
远程仓库已有一个 `Initial commit`，本地的初始提交与之没有共同祖先，Git 拒绝推送。

### 解决方案
使用 `--force` 强制推送覆盖远程分支。

---

## 错误二：Git 凭证管理器失效

### 错误信息
```
git: 'credential-manager-core' is not a git command.
```

### 原因
Git 配置中 `credential.helper` 设置为 `manager-core`，但系统找不到该程序。

### 解决方案
1. 切换为标准的 `manager`：
   ```
   git config --global credential.helper manager
   ```
2. 改用 SSH 方式推送（更稳定）：
   ```
   git remote set-url origin git@github.com:username/repo.git
   ```

---

## 错误三：推送包超过 GitHub 大小限制

### 错误信息
```
remote: fatal: pack exceeds maximum allowed size (2.00 GiB)
fatal: sha1 file '<stdout>' write error: Broken pipe
error: remote unpack failed: index-pack failed
! [remote rejected] main -> main (failed)
```

### 原因
项目包含大量纹理文件（.tga / .tif），本地仓库总大小约 **3.36 GiB**，超过了 GitHub 单次推送包 **2 GiB** 的限制。

**主要的大文件：**
- `Knight_05_Sword_Normals.tga` — ~201 MB
- `GR_Asphalt1_AS.tif` — ~67 MB
- 以及其他大量 64 MB 级别的纹理

### 解决方案
创建一个只包含脚本文件的新分支，将仓库瘦身：
```bash
# 创建纯净分支
git checkout --orphan scripts-only

# 只添加脚本和配置
git add Assets/Scripts/ .gitignore

# 提交并强制推送到远程 main
git push --force origin scripts-only:main
```

---

## 经验教训

| 要点 | 说明 |
|------|------|
| 大文件管理 | Unity 项目的 .tga / .tif / .fbx 等资源文件体积巨大，不应直接提交到 Git |
| 推荐方案 | 使用 **Git LFS** 管理大文件，或将代码和资源拆分到不同仓库 |
| 认证方式 | HTTPS + Token 可能受网络环境影响，**SSH** 更稳定可靠 |
| GitHub 限制 | 单文件 ≤ 100 MB，单次推送 ≤ 2 GiB，仓库推荐 ≤ 1 GiB |
| 企业项目 | 十几个 GB 的项目建议自建 Git 服务器（如 Gitea、GitLab）|

## 后续建议

1. **仅推送代码**：后续只将 `Assets/Scripts/` 下的脚本推送到 GitHub
2. **使用 Git LFS**：如果需要管理纹理等大资源，配置 `.gitattributes` 启用 LFS
3. **拆分仓库**：将项目拆分为代码仓库和资源仓库，通过子模块管理
