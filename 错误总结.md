# 玩家自动旋转问题总结

## 问题描述

玩家角色在移动时会不受控制地**凭空自动旋转**，即使不按方向键也会自己转圈。

---

## 原因分析

角色动画文件（如 `run_forward.anim`、`walk_front.anim`）中**包含了旋转数据**，动画播放时会直接写入 `Transform.rotation`，导致 Animator 不断修改角色的朝向，与脚本控制的旋转产生冲突。

```
动画写入 Transform.rotation  →  角色自动旋转
脚本调用 rb.MoveRotation()   →  也在控制旋转
                ↓
        两方冲突 → 角色失控
```

---

## 解决方案

### 核心思路

通过 **Rigidbody 的 Freeze Rotation** 阻挡动画对旋转的修改，同时保留脚本的转向控制。

### 具体步骤

1. **在 Inspector 中冻结 Rigidbody 的 Y 轴旋转**
   - 选中角色 GameObject
   - 找到 Rigidbody 组件 → **Constraints**
   - 勾选 `Freeze Rotation Y`

2. **转向全部使用 `rb.MoveRotation()`**
   - 不要用 `transform.rotation` 直接赋值
   - 使用 `Rigidbody.MoveRotation()` 进行平滑旋转

### 原理说明

| 操作 | 方式 | 受 Freeze 影响？ |
|------|------|:---:|
| `rb.MoveRotation()` | 走 Rigidbody 内部通道 | ❌ 不受影响 |
| 动画修改 Transform | 直接写入 Transform | ✅ 被冻结阻止 |
| 物理碰撞反作用力 | 外部力 | ✅ 被冻结阻止 |

`rb.MoveRotation` 是 Rigidbody 自己的 API，不会绕过 Freeze 约束；而**动画、碰撞、其他脚本直接改 Transform** 的旋转，都会被 Freeze 挡住。

---

## 最佳实践

### Rigidbody 约束标准配置

```
冻结 X 轴 — 防止角色前后倾倒
冻结 Z 轴 — 防止角色左右歪斜
冻结 Y 轴 — 挡住动画/物理的幽灵旋转
        ↓
转向全部用 rb.MoveRotation()
```

### 代码示例

```csharp
// ✅ 正确：用 Rigidbody 控制旋转
private void FixedUpdate()
{
    if (direction.magnitude > 0.01f)
    {
        Quaternion targetRotation = Quaternion.LookRotation(direction);
        rb.MoveRotation(Quaternion.Slerp(rb.rotation, targetRotation, rotateSpeed * Time.fixedDeltaTime));
        rb.MovePosition(rb.position + transform.forward * moveSpeed * Time.fixedDeltaTime);
    }
}
```

---

## 总结

| 控制方式 | 能否冻结 Y 轴？ | 说明 |
|---------|:-----------:|------|
| `rb.MoveRotation` 控制转向 | ✅ 可以冻结 | MoveRotation 走自有通道，不受限制 |
| `transform.rotation` 控制转向 | ❌ 不能冻结 | 冻结后转向也会失效 |

遇到角色无故自动旋转，优先检查 **动画是否写入了旋转**，然后通过 **Freeze Rotation Y + MoveRotation** 解决。
